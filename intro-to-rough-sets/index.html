Introduction

In computer science, a rough set, first described by Polish computer scientist Zdzisław I. Pawlak, is a formal approximation of a crisp set (i.e., conventional set) in terms of a pair of sets which give the lower and the upper approximation of the original set. Rough sets constitutes a sound basis for Knowledge Discovery in Databases (KDD). It offers mathematical tools to discover patterns hidden in data, feature selection, feature extraction, data reduction, decision rule generation etc.

Basic Concepts of Rough Sets
  • Information/Decision Systems
  • Indiscernibility
  • Set Approximations

Information System (IS)

IS = (U, A). U is a non-empty finite set of objects, A is a non-empty finite set of attributes such that a: U → Va for every a ∈ A. Va is called the value set of a.

Decision System (DS)

DS = (U, A ∪ D). Elements of A are called conditional attributes, D contains one or more decision attributes. In the below DS d is a1 OR a2

Let U = {O1, O2, O3, O4, O5} be the universal set of the objects.
Any set X ⊆ U is called a concept. ‘Attribute-1’ induces a relation R on U thus dividing it into two concepts C1 = {O1, O2, O4} and C2 = {O3, O5}. These concepts C1 and C2 are clearly explainable by the knowledge using ‘Attribute-1’. Now consider a concept C = {O1, O4} which can not be explained by the knowledge. Hence, we try to see if we can approximately explain the concepts using Rough sets.

Now, the concept C = {O1, O4} can not be explained by the knowledge. To explain the concept, we define lower and upper approximations to the concept. The lower approximation contains the elements that can be classified with complete certainty as the members of C. The upper approximation includes the elements that can be classified as the members of C with any certainty. Any concept whose lower and upper approximations are equal is called an R-definable concept.
